Overview
tome is a Rust CLI that discovers skills (reusable AI instructions in SKILL.md files) from multiple sources, consolidates them into a central library, and distributes them to any AI coding tool — Claude Code, Codex, Copilot, Cursor, Antigravity, and more.

The project is a Rust workspace (edition 2024) producing two binaries: tome (the full CLI) and tome-mcp (a standalone MCP server). All domain logic lives in crates/tome as a library, with a thin main.rs that calls tome::run().

Source Types
  • ClaudePlugins — reads installed_plugins.json (v1 & v2)
  • Directory — flat walkdir scan of any folder
First source wins on name conflicts. Exclusion list applied.
Distribution Methods
  • Symlink — creates links in target's skills dir → library
  • Mcp — writes tome entry into target's .mcp.json
Targets configured as BTreeMap<String, TargetConfig> (data-driven).

Key insight: Sources are read-only inputs. The library owns canonical copies (real directories, not symlinks). Targets receive symlinks into the library. This two-tier design means no data loss risk — originals are never moved.
CLI Commands
tome init
Interactive wizard. Auto-discovers sources, configures targets, runs first sync.
tome sync
Discover → Consolidate → Distribute → Cleanup. Idempotent.
tome list
Table-formatted skill listing. Supports --json output.
tome status
Read-only summary: library size, sources, targets, health.
tome doctor
Diagnoses orphans, missing manifests, broken symlinks. Optional repair.
tome serve
MCP server over stdio. Exposes list_skills and read_skill.
Global flags: --dry-run (skip writes, report what would change), --quiet (suppress non-error output), --verbose (per-stage details). Also: tome config --path for config location.
Tool Support Landscape

AI coding tools have up to 7 configuration layers. Skills and MCP are the two truly portable layers — everything else is tool-specific.

Layer Purpose Portable? Supported By
Skills Reusable instructions, on-demand Yes (SKILL.md standard) Claude, Codex, Copilot, Cursor, Antigravity, Amp, Goose, 20+ tools
Rules Always-on project conventions Partial All tools (different filenames/formats)
Memory Learned context across sessions No Claude, Codex, Cursor, Windsurf, Copilot
Hooks Lifecycle event handlers No Claude (12 events), Codex, Windsurf, Amp
Agents Isolated subagents with custom tools No Claude, Codex, Copilot, Cursor, Antigravity
Plugins Bundles of skills + agents + hooks No Claude, Cursor, Amp
MCP Servers External tool integrations Yes (open standard) All major tools
