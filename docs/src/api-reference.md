## Key public types

For a v1.0 GUI / library consumer, the most important types to know about:

- **[`SyncReport`](../api/tome/struct.SyncReport.html)** ‚Äî return shape of the full `sync()` pipeline (reconcile ‚Üí discover ‚Üí consolidate ‚Üí distribute ‚Üí cleanup ‚Üí save). Primary data source for any "what happened this sync" surface.
- **[`reconcile::ReconcileReport`](../api/tome/reconcile/struct.ReconcileReport.html)** ‚Äî outcome of one reconcile pass (Match / Drift / Vanished / Missing classifications plus edit-in-library user decisions).
- **[`marketplace::MarketplaceAdapter`](../api/tome/marketplace/trait.MarketplaceAdapter.html)** ‚Äî pluggable trait for managed-skill install/update/availability. Two production implementations ship (`ClaudeMarketplaceAdapter`, `GitAdapter`); third-party adapters can implement the trait directly (sealing is tracked as a v1.0 follow-up in [#518](https://github.com/MartinP7r/tome/issues/518)).
- **[`CleanupResult`](../api/tome/cleanup/struct.CleanupResult.html)** ‚Äî bucket-by-bucket cleanup outcomes (removed-from-config / missing-from-disk / now-in-exclude-list). Accessors are read-only; the renderer owns the user-facing surface.
- **[`status::StatusReport`](../api/tome/status/struct.StatusReport.html)** + **[`status::DirectoryStatus`](../api/tome/status/struct.DirectoryStatus.html)** ‚Äî `tome status --json` shape. Since v0.12.0, `DirectoryStatus.role` is the typed `DirectoryRole` enum (with the human-readable description in a separate `role_description` field).
- **[`doctor::DoctorReport`](../api/tome/doctor/struct.DoctorReport.html)** ‚Äî `tome doctor --json` shape, including `IssueCategory` (Library / Directory / Config / Foreign-symlink) groupings.

The architecture narrative ‚Äî how these types fit together, why the library is canonical, how reconcile interacts with the manifest and lockfile ‚Äî lives in **[Architecture](architecture.html)**.

## Newtypes

Domain-specific wrappers with validation at construction:

- **[`SkillName`](../api/tome/discover/struct.SkillName.html)** ‚Äî validated skill identifier
- **[`config::DirectoryName`](../api/tome/config/struct.DirectoryName.html)** ‚Äî validated directory entry name
- **[`validation::ContentHash`](../api/tome/validation/struct.ContentHash.html)** ‚Äî SHA-256 of a skill directory, normalized to lowercase hex
- **[`paths::TomePaths`](../api/tome/paths/struct.TomePaths.html)** ‚Äî bundle of `tome_home` / `library_dir` / `config_dir` to prevent parameter-order bugs
