    Create `crates/tome/src/remove.rs` following the plan/render/execute pattern from eject.rs:

    ```rust
    //! Remove a directory entry from config and clean up all artifacts.
    //!
    //! Cleanup order (per D-12, pitfall 6):
    //! 1. Remove symlinks from distribution directories
    //! 2. Remove library entries for skills from this directory
    //! 3. Remove manifest entries
    //! 4. Remove cached git repo (if git type)
    //! 5. Remove directory entry from config
    //! 6. Regenerate lockfile
    ```

    Structs:
    - `RemovePlan` — what will be removed: directory name, directory config (clone for display), skills from this directory (Vec<String> from manifest scan), symlinks to remove (Vec<PathBuf>), git cache path (Option<PathBuf>), library paths to remove (Vec<PathBuf>).
    - `RemoveResult` — counts: symlinks_removed, library_entries_removed, config_updated, cache_removed.

    Functions:

    1. **`pub(crate) fn plan(name: &str, config: &Config, paths: &TomePaths, manifest: &Manifest) -> Result<RemovePlan>`**
       - Validate directory exists in config: `config.directories.get(name)` or bail with "directory '{}' not found in config"
       - Scan manifest for skills whose `source_name == name` — these are the skills to remove
       - For each skill, check all distribution directories for symlinks pointing into library
       - If git type, compute cache path via `git::repo_cache_dir(paths.repos_dir(), &dir_config.path.to_string_lossy())`
       - Return RemovePlan

    2. **`pub(crate) fn render_plan(plan: &RemovePlan)`**
       - Print what will be removed using `console::style`:
         - "Directory: {name}" with path
         - "Skills to remove from library: {count}" with names listed
         - "Symlinks to remove: {count}"
         - "Git cache to remove: {path}" (if git type)
         - "Config entry will be removed"

    3. **`pub(crate) fn execute(plan: &RemovePlan, config: &mut Config, manifest: &mut Manifest, paths: &TomePaths, dry_run: bool) -> Result<RemoveResult>`**
       - Follow cleanup order:
         1. Remove symlinks (fs::remove_file, collect warnings on failure)
         2. Remove library dirs (fs::remove_dir_all for each skill)
         3. Remove manifest entries (manifest.remove for each skill)
         4. Remove git cache (fs::remove_dir_all if Some)
         5. Remove from config.directories BTreeMap
       - Return RemoveResult with counts
       - In dry_run mode, count but don't act

    **lib.rs — Add module and dispatch:**

    Add `pub(crate) mod remove;` to module declarations (between `relocate` and `skill` alphabetically).

    Add match arm in `run()` for `Command::Remove { name, force }`:
    ```rust
    Command::Remove { name, force } => {
        let manifest = manifest::load(paths.config_dir())?;
        let plan = remove::plan(&name, &config, &paths, &manifest)?;
        remove::render_plan(&plan);

        if plan.is_empty() {
            println!("Nothing to remove.");
            return Ok(());
        }

        if cli.dry_run {
            println!("\n{}", style("Dry run — no changes made.").yellow());
            return Ok(());
        }

        // Confirmation per D-13: TTY = interactive, non-TTY = auto with warning
        if !force {
            if std::io::stdin().is_terminal() {
                let confirmed = dialoguer::Confirm::new()
                    .with_prompt(format!("Remove directory '{}'?", name))
                    .default(false)
                    .interact()?;
                if !confirmed {
                    println!("Aborted.");
                    return Ok(());
                }
            } else if !cli.no_input {
                eprintln!("warning: removing directory '{}' in non-interactive mode", name);
            } else {
                anyhow::bail!(
                    "tome remove requires confirmation — use --force in non-interactive mode"
                );
            }
        }

        let mut config = config.clone();
        let mut manifest = manifest;
        let result = remove::execute(&plan, &mut config, &mut manifest, &paths, false)?;

        // Save updated config
        config::save(&config, &paths.config_path())?;
        // Save updated manifest
        manifest::save(&manifest, paths.config_dir())?;
        // Regenerate lockfile
        let mut warnings = Vec::new();
        let skills = discover::discover_all(&config, &BTreeMap::new(), &mut warnings)?;
        let lockfile = lockfile::generate(&manifest, &skills);
        lockfile::save(&lockfile, paths.config_dir())?;

        println!(
            "\n{} Removed directory '{}': {} library entries, {} symlinks{}",
            style("✓").green(),
            name,
            result.library_entries_removed,
            result.symlinks_removed,
            if result.cache_removed { ", git cache cleaned" } else { "" }
        );
    },
    ```

    NOTE: The executor must verify that `config::save`, `manifest::save`, and `lockfile::save` functions exist with these signatures. If `Config` doesn't have a `save` function, create one following the atomic temp+rename pattern from machine.rs. The executor should check config.rs, manifest.rs, and lockfile.rs for existing save functions.
