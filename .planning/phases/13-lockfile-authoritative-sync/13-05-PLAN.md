Plan 13-05 needs the integration test to opt into `test-support` so `tome::marketplace::testing` resolves. The way to do this in the integration test is via `[dev-dependencies] tome = { path = ".", features = ["test-support"] }` in the workspace `Cargo.toml`. **However**, the crate's own integration tests automatically have access to `cfg(test)` items, so they ALREADY see `pub mod testing` (which is gated `cfg(any(test, feature = "test-support"))`). No Cargo.toml change is needed unless we discover during execution that integration tests in `tests/*.rs` don't satisfy `cfg(test)` for the *library* target.

**Empirical fact about Rust integration tests:** `tests/*.rs` is compiled as a separate crate target. `cfg(test)` is set for that target, but NOT for the dependency `tome` it links against. The `tome` library it links against is built without `cfg(test)`. So `pub mod testing` in `marketplace.rs` is NOT visible from `tests/cli_sync_reconcile.rs` unless `feature = "test-support"` is enabled in the test target's link.

**Resolution:** Add `tome = { path = ".", features = ["test-support"] }` to `[dev-dependencies]` in `crates/tome/Cargo.toml`. This is unusual (a crate depending on itself in dev-dependencies) but it's the standard Rust idiom for feature-gating test-support across integration tests. Cargo handles the self-dep correctly because dev-dependencies don't participate in cyclic resolution.

Reference: https://doc.rust-lang.org/cargo/reference/features.html — "Features for dev-only".

//! Integration tests for Phase 13 (RECON-01..05) reconcile flow.
//!
//! These tests exercise the `tome sync` binary end-to-end via assert_cmd,
//! covering the non-interactive flow paths only. Per RESEARCH Pitfall 6,
//! `dialoguer::Select` cannot be driven from `assert_cmd::write_stdin` — the
//! interactive consent / edit-in-library prompts are covered by
//! `crates/tome/src/reconcile.rs::tests` (Plan 13-03), which exercise
//! `MockMarketplaceAdapter` directly.
//!
//! What these tests cover:
//! - Summary line regex on stdout (RECON-01 / D-02 / D-04).
//! - Vanished entry → distribution still symlinks preserved library copy
//!   (RECON-04 anchor).
//! - `--no-install` skips apply unconditionally; exit zero (RECON-02 / D-09).
//! - `--no-input` + edit-in-library detected → skip-with-warning + exit zero
//!   (RECON-05 / D-16).
//! - Missing `claude` binary + `claude-plugins` directory → D-20 error +
//!   non-zero exit.
//! - `auto_install_plugins` round-trip across syncs (RECON-02 persistence).

use std::collections::BTreeMap;
use std::path::{Path, PathBuf};
use std::process::Command;

use anyhow::Result;
use assert_cmd::prelude::*;
use predicates::prelude::*;

/// Build a tome home + config + library at `<tmp>/{tome_home,library,etc}`.
struct Fixture {
    _tmp: TempDir,
    tome_home: PathBuf,
    library_dir: PathBuf,
    config_path: PathBuf,
    machine_path: PathBuf,
    dist_dir: PathBuf,
}

impl Fixture {
    /// Empty library + a single distribution dir + a synthetic claude-plugins
    /// directory pointing at a path that doesn't contain a real claude install.
    fn new() -> Result<Self> {
        let tmp = TempDir::new()?;
        let root = tmp.path().to_path_buf();
        let tome_home = root.join("tome_home");
        let library_dir = tome_home.join("library");
        let config_path = tome_home.join("tome.toml");
        let machine_path = root.join("machine.toml");
        let dist_dir = root.join("dist");

        std::fs::create_dir_all(&library_dir)?;
        std::fs::create_dir_all(&dist_dir)?;
        std::fs::write(&machine_path, "")?;

        Ok(Fixture {
            _tmp: tmp,
            tome_home,
            library_dir,
            config_path,
            machine_path,
            dist_dir,
        })
    }

    /// Write a `tome.toml` with one local `directory` (source) + one
    /// distribution dir. No claude-plugins entry — for tests that don't need
    /// the adapter to fire.
    fn write_local_only_config(&self, source_dir: &Path) -> Result<()> {
        let toml = format!(
r#"[directories.local-skills]
type = "directory"
role = "source"
path = "{src}"

[directories.dist]
type = "directory"
role = "distribution"
path = "{dist}"
"#,
            src = source_dir.display(),
            dist = self.dist_dir.display(),
        );
        std::fs::write(&self.config_path, toml)?;
        Ok(())
    }

    /// Write a `tome.toml` with a `claude-plugins` directory (no real claude
    /// install) + a distribution dir. Used for D-20 "claude binary missing"
    /// test.
    fn write_claude_plugins_config(&self) -> Result<()> {
        let claude_path = self.tome_home.join("claude_pseudo");
        std::fs::create_dir_all(&claude_path)?;
        let toml = format!(
r#"[directories.cp]
type = "claude-plugins"
role = "source"
path = "{path}"

[directories.dist]
type = "directory"
role = "distribution"
path = "{dist}"
"#,
            path = claude_path.display(),
            dist = self.dist_dir.display(),
        );
        std::fs::write(&self.config_path, toml)?;
        Ok(())
    }

    /// Run `tome sync` with the given extra args. Returns the assert_cmd
    /// output for the caller to inspect.
    fn run_sync(&self, args: &[&str]) -> Command {
        let mut cmd = Command::cargo_bin("tome").expect("tome binary builds");
        cmd.arg("--config").arg(&self.config_path)
            .arg("--tome-home").arg(&self.tome_home)
            .arg("--machine").arg(&self.machine_path)
            .arg("sync")
            .args(args);
        cmd
    }

    /// Same but with PATH cleared so `claude` is not findable. Used for D-20.
    fn run_sync_no_claude(&self, args: &[&str]) -> Command {
        let mut cmd = self.run_sync(args);
        cmd.env_clear()
            // Preserve HOME so dirs::home_dir() works
            .env("HOME", std::env::var_os("HOME").unwrap_or_default())
            // PATH = empty (no claude binary findable)
            .env("PATH", "");
        cmd
    }
}

/// Write a minimal SKILL.md tree at `<dir>/<skill_name>/SKILL.md`.
fn write_skill(dir: &Path, name: &str, body: &str) -> Result<PathBuf> {
    let skill_dir = dir.join(name);
    std::fs::create_dir_all(&skill_dir)?;
    let frontmatter = format!(
        "---\nname: {name}\ndescription: Test skill {name}\n---\n\n{body}\n"
    );
    std::fs::write(skill_dir.join("SKILL.md"), frontmatter)?;
    Ok(skill_dir)
}
