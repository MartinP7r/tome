    - crates/tome/src/config.rs (new types from Plan 01 -- DirectoryName, DirectoryConfig, discovery_dirs)
    - crates/tome/src/manifest.rs (SkillEntry fields, especially source_name and source_path)
  </read_first>
  <action>
    **discover.rs changes:**

    1. Update imports: replace `use crate::config::{Config, Source, SourceType}` with `use crate::config::{Config, DirectoryConfig, DirectoryName, DirectoryType, DirectoryRole}`.

    2. Update `discover_all()` signature and body:
       - Currently iterates `config.sources` (Vec<Source>)
       - Change to iterate `config.discovery_dirs()` which yields `(&DirectoryName, &DirectoryConfig)`
       - For each directory, dispatch by `dir_config.directory_type`:
         - `DirectoryType::ClaudePlugins` -> call discover_claude_plugins (existing logic)
         - `DirectoryType::Directory` -> call discover_directory (existing logic)
         - `DirectoryType::Git` -> same as Directory (git resolution happens pre-discovery in Phase 2)
       - Populate `DiscoveredSkill.source_name` from `dir_name.as_str().to_string()` (was `source.name.clone()`)
       - Deduplication: first-seen-wins is unchanged; BTreeMap iteration order now provides alphabetical priority (PIPE-05)

    3. Update `DiscoveredSkill.origin` determination:
       - Currently: `SourceType::ClaudePlugins` -> `SkillOrigin::Managed { provenance }`
       - New: `dir_config.role() == DirectoryRole::Managed` -> `SkillOrigin::Managed { provenance }`
       - All other roles -> `SkillOrigin::Local`

    4. Update internal helper functions that take `&Source` to take `(&DirectoryName, &DirectoryConfig)` or `(&str, &Path, &DirectoryType)` as appropriate.

    5. Keep `SkillName`, `DiscoveredSkill`, `SkillOrigin` types unchanged. Keep `source_name` field as `String`.

    6. Update unit tests: replace any Source/SourceType construction with DirectoryConfig/DirectoryName construction.

    **library.rs changes:**

    1. Update consolidation strategy selection:
       - Currently: checks `SkillOrigin::Managed` to decide symlink vs copy
       - This logic is correct and doesn't change -- Managed origin still means symlink consolidation, Local origin means copy. The SkillOrigin is already set correctly by updated discover.rs.
       - Update any imports that reference `Source`, `SourceType`, `TargetName`, `TargetConfig` to use new types.

    2. If library.rs references `config.sources` or `config.targets` anywhere, replace with `config.discovery_dirs()` / `config.distribution_dirs()` as appropriate.

    3. Update unit tests to use new config types.
  </action>

    3. Rename `distribute_to_target()` to `distribute_to_directory()`:
       - Old signature: `fn distribute_to_target(library_dir, target_name: &str, target: &TargetConfig, manifest, machine_prefs, source_paths: &[PathBuf], dry_run, force)`
       - New signature: `pub fn distribute_to_directory(library_dir: &Path, dir_name: &DirectoryName, dir_config: &DirectoryConfig, manifest: &Manifest, machine_prefs: &MachinePrefs, dry_run: bool, force: bool) -> Result<DistributeResult>`

         ```rust
         if let Some(manifest_entry) = manifest.get(skill_name_str) {
             if manifest_entry.source_name == dir_name.as_str() {
                 result.skipped_managed += 1;
                 continue;
             }
         }
         ```
       - This checks: "was this skill discovered from the same directory we're distributing to?" If yes, skip.

       - Remove `source_paths` parameter
       - Use `dir_name` and `dir_config.path` instead of `target_name` and `skills_dir`
