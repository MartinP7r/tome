    Add a new section at the top for v0.6 (unreleased):
    ```markdown
    ## [Unreleased] - v0.6.0

    ### Breaking Changes

    **Unified directory model replaces sources/targets**

    tome v0.6 replaces the separate `[[sources]]` and `[targets.*]` config with a unified `[directories.*]` model. Each directory now declares a `type` and `role`.

    **Before (v0.5.x):**
    ```toml
    [[sources]]
    name = "claude-plugins"
    path = "~/.claude/plugins"
    type = "claude-plugins"

    [[sources]]
    name = "claude-skills"
    path = "~/.claude/skills"
    type = "directory"

    [targets.claude]
    enabled = true
    method = "symlink"
    skills_dir = "~/.claude/skills"
    ```

    **After (v0.6):**
    ```toml
    [directories.claude-plugins]
    path = "~/.claude/plugins"
    type = "claude-plugins"
    role = "managed"

    [directories.claude-skills]
    path = "~/.claude/skills"
    # type = "directory" (default), role = "synced" (default)
    ```

    **Migration rule:** Each source becomes a `[directories.*]` entry. Each target becomes a `[directories.*]` entry. If a source and target pointed to the same path, merge them into a single entry with `role = "synced"`.

    **Roles:**
    - `managed` — Read-only, owned by package manager (use with `type = "claude-plugins"`)
    - `synced` — Skills discovered here AND distributed here (default)
    - `source` — Skills discovered here, not distributed here
    - `target` — Skills distributed here, not discovered here

    **Also changed:**
    - `machine.toml`: `disabled_targets` renamed to `disabled_directories`
    - Run `tome init` to regenerate your config from scratch (recommended)
    ```
