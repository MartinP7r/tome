## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01-01 | Tampering | `LockfileState::classify` reading `tome.lock` mid-rename by external CLI | accept | Reads are atomic (CLI uses temp+rename); worst case is reading the old version, which the file watcher (plan 26-06) refetches. No partial reads. |
| T-26-01-02 | InformationDisclosure | `MachinePrefsSummary` exposes disabled counts to the webview | accept | Counts are scalar integers (no PII); same trust boundary as `tome status` JSON output. |
| T-26-01-03 | DoS | Webview re-renders on every prop change when StatusReport is large | mitigate | React keys + memoization on `<DirectoryTable>` rows; `useStatus` only refetches on explicit `refetch()` (event subscriptions land in 26-06, which adds debounce). |
| T-26-01-04 | EoP | New cross-boundary types not specta-gated | mitigate | `LockfileState` + `MachinePrefsSummary` derive `#[cfg_attr(feature = "bindings", derive(specta::Type))]`; CI freshness gate (Phase 25 `git diff --exit-code` on `bindings.ts`) catches drift. |
