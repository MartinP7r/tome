# Phase 3 — UI Design Contract

> Visual and interaction contract for the browse TUI polish and CLI command output. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rust CLI + ratatui TUI) |
| Preset | not applicable |
| Component library | ratatui 0.30 + crossterm 0.29 |
| Icon library | none (Unicode box-drawing + symbols) |
| Font | Terminal default (monospace, user-controlled) |

---

## Spacing Scale

Terminal UI uses character cells, not pixels. All spacing is in **character units** (cols/rows).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 col | Inline padding between label and value |
| sm | 2 col | Column gaps in table output |
| md | 1 row | Vertical gap between sections |
| lg | 2 row | Vertical gap between major blocks (e.g., metadata to actions in detail view) |

Exceptions: Status bar is always exactly 1 row. Header + separator = 2 rows fixed.

---

## Typography

Terminal UI has no font-size control. "Typography" maps to **text decoration** in ratatui.

| Role | Style | Usage |
|------|-------|-------|
| Body | `Style::default()` (no modifier) | Skill names, paths, source names, preview body text |
| Label | `Style::default().fg(theme.muted)` | Metadata labels ("Source:", "Type:", "Path:", "Synced:") |
| Heading | `Modifier::BOLD` + `theme.accent` fg | Column headers (SKILL, SOURCE, PATH), group headers, preview markdown `#` headers |
| Emphasis | `Modifier::BOLD` | Selected row, status bar count badge, `**bold**` in markdown preview |
| Code | `theme.code_fg` fg | Inline `` `code` `` spans in markdown preview |
| Italic | `Modifier::ITALIC` | `*italic*` in markdown preview |

## Color

### Terminal-Adaptive Theme (D-10)

Detection: Check `$COLORFGBG` env var. Parse `"fg;bg"` format. If `bg >= 9` (excluding 8 which is dark gray), use light palette. Otherwise default to dark palette.

**Dark palette** (default):

| Role | ANSI Color | ratatui Value | Usage |
|------|------------|---------------|-------|
| Dominant (60%) | Terminal default bg | `Color::Reset` | All background surfaces |
| Secondary (30%) | Dark gray | `Color::DarkGray` | Status bar bg, separator lines, metadata labels fg, selected row bg |
| Accent (10%) | Cyan | `Color::Cyan` | Column headers, detail title, action pointer, status count badge bg, active mode badge |
| Alert | Yellow | `Color::Yellow` | Group headers, search input display, match highlight fg |
| Muted | Gray | `Color::Gray` | Status bar hints, inactive text |
| Destructive | Red | `Color::Red` | Error messages, failed operations |
| Success | Green | `Color::Green` | Confirmation messages after add/reassign/fork |
| Code | Magenta | `Color::Magenta` | Inline code spans in markdown preview |

**Light palette** (when detected):

| Role | ANSI Color | ratatui Value | Usage |
|------|------------|---------------|-------|
| Dominant (60%) | Terminal default bg | `Color::Reset` | All background surfaces |
| Secondary (30%) | Indexed 254 (near-white gray) | `Color::Indexed(254)` | Status bar bg, separator lines |
| Accent (10%) | Dark cyan | `Color::Indexed(30)` | Column headers, detail title, action pointer, status count badge bg |
| Alert | Dark yellow | `Color::Indexed(136)` | Group headers, search input display, match highlight fg |
| Muted | Indexed 243 (medium gray) | `Color::Indexed(243)` | Status bar hints, inactive text |
| Destructive | Dark red | `Color::Indexed(124)` | Error messages |
| Success | Dark green | `Color::Indexed(28)` | Confirmation messages |
| Code | Dark magenta | `Color::Indexed(133)` | Inline code spans |

Accent reserved for: Column header text, detail view title, action list selection indicator (`>`), status bar mode badge background, status count badge background. Never used for body text or fill areas.

## Layout Contract

### Normal Mode (existing, modified)

```
+----------------------------------------------------------+
| SKILL          SOURCE       PATH                          |  <- header (1 row)
| ────────────────────────────────────────────────────────  |  <- separator (1 row)
| skill-name-1   source-a     ~/.tome/lib/skill-name-1   |│|  <- body left 45% + scrollbar
| skill-name-2   source-b     ~/.tome/lib/skill-name-2   | |
| ...                                                     | |  <- body right 55% = preview
|                                                          | |
| 12 skills | / search  s sort:name  tab group  ? help  q |  <- status bar (1 row)
+----------------------------------------------------------+
```

- Left panel: 45% width (unchanged)
- Right panel: 55% width (unchanged)
- Scrollbar: Rendered inside left panel right edge, 1 col wide. Only visible when `filtered_indices.len() > visible_height`. Uses `Scrollbar::new(ScrollbarOrientation::VerticalRight)` with `begin_symbol(None)` and `end_symbol(None)`.

### Help Overlay (new — D-14)

Triggered by `?` key in Normal mode. Rendered as a centered popup over the current view.

```
+--------------------------------------+
|            Keyboard Shortcuts         |
|--------------------------------------|
|  j / Down      Move down             |
|  k / Up        Move up               |
|  g             Jump to top            |
|  G             Jump to bottom         |
|  Ctrl+d        Half page down         |
|  Ctrl+u        Half page up           |
|  /             Search                 |
|  Esc           Clear search / back    |
|  s             Cycle sort mode        |
|  Tab           Toggle source grouping |
|  Enter         Skill detail           |
|  q             Quit                   |
|  ?             This help              |
|                                       |
|          Press any key to close       |
+--------------------------------------+
```

- Size: 40 cols wide, 18 rows tall (fixed)
- Centered horizontally and vertically in frame
- Border: `Borders::ALL` with `theme.accent` border style
- Title: "Keyboard Shortcuts" bold
- Background: `Color::Reset` (clears underlying content)
- Dismiss: Any keypress returns to previous mode

### Detail Mode (existing, unchanged layout)

- Left panel: 40% width
- Right panel: 60% width
- Metadata section: 6 rows fixed
- Actions list: remaining vertical space

## Fuzzy Match Highlighting (D-12)

| Property | Value |
|----------|-------|
| Highlighted chars style | `Style::default().fg(theme.alert).add_modifier(Modifier::BOLD)` |
| Scope | Skill name column only (not source, not path, not preview) |
| Index source | `nucleo_matcher::pattern::Atom::indices()` returning `Vec<u32>` char positions |
| Haystack for indices | Name string only (not composite `"{name} {source}"`) to avoid off-by-one |
| Non-matching chars | Default body style |

Implementation: Build `Line` from `Vec<Span>`, one per character. Characters at matched indices get highlight style; others get default style.

---

## Markdown Preview Rendering (D-11)

Line-by-line parser. No dependency on pulldown-cmark.

| Pattern | Detection | Rendered Style |
|---------|-----------|----------------|
| `# heading` | `line.strip_prefix("# ")` | `Modifier::BOLD` + `theme.accent` fg |
| `## heading` | `line.strip_prefix("## ")` | `Modifier::BOLD` + `theme.accent` fg |
| `### heading` | `line.strip_prefix("### ")` | `Modifier::BOLD` + `theme.accent` fg |
| `---` | `line.starts_with("---")` | 40x `─` in `theme.muted` fg |
| `**bold**` | Inline delimiter scan | `Modifier::BOLD` |
| `*italic*` | Inline delimiter scan (single `*`) | `Modifier::ITALIC` |
| `` `code` `` | Inline backtick pairs | `theme.code_fg` fg |
| Plain text | Default | Body style |

Rendering order for inline: scan left-to-right for delimiter pairs. Nested delimiters (e.g., `**bold *italic***`) are not supported — keep the parser simple. Lists (`-`, `*`, numbered) render as plain text (no transformation). Tables render as plain text.

## Scrollbar (D-13)

| Property | Value |
|----------|-------|
| Widget | `ratatui::widgets::Scrollbar` + `ScrollbarState` |
| Orientation | `ScrollbarOrientation::VerticalRight` |
| Begin/end symbols | `None` / `None` (clean track) |
| Track symbol | Default (ratatui standard) |
| Thumb symbol | Default (ratatui standard) |
| Visibility | Only when `filtered_indices.len() > visible_height` |
| State | `ScrollbarState::new(total_items).position(scroll_offset)` |
| Render area | Same rect as left body panel (scrollbar overlays rightmost col) |

## Copywriting Contract

### CLI Commands

| Element | Copy |
|---------|------|
| `tome add` success | `Added directory '{name}' (git: {url})` |
| `tome add` dry-run | `Would add directory '{name}' (git: {url})` |
| `tome add` duplicate error | `Error: directory '{name}' already exists in config` |
| `tome add` invalid URL | `Error: could not extract repository name from '{url}'. Use --name to specify manually.` |
| `tome reassign` success | `Reassigned '{skill}' from '{old_dir}' to '{new_dir}'` |
| `tome reassign` skill-not-found | `Error: skill '{name}' not found in library` |
| `tome reassign` dir-not-found | `Error: directory '{name}' not found in config` |
| `tome fork` confirmation | `Fork '{skill}' to '{dir}'? This copies skill files to the target directory. [y/N]` |
| `tome fork` success | `Forked '{skill}' to '{dir}' (local copy created)` |
| `tome fork` dry-run | `Would fork '{skill}' to '{dir}' (copy {n} files)` |

### Browse TUI

| Element | Copy |
|---------|------|
| Empty state (Normal mode, no skills) | Preview: `"No skills discovered."` Title: `"Preview"` |
| Empty state (Search, no matches) | Preview: `"No matching skill."` Title: `"Preview"` |
| SKILL.md empty | `"[SKILL.md is empty]"` (existing) |
| SKILL.md read error | `"[failed to read {path}: {err}]"` (existing) |
| Help overlay title | `"Keyboard Shortcuts"` |
| Help overlay footer | `"Press any key to close"` |
| Status bar (Normal) | `" {n} skills | / search  s sort:{mode}  tab group  ? help  q quit"` |
| Status bar (Detail) | `" Detail | j/k select  Enter run action  esc back"` |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| `tome fork` | Interactive `[y/N]` prompt via `dialoguer::Confirm`. Skipped with `--force`. (D-09) |
| `tome reassign` | No confirmation (metadata-only change). (D-09) |
| `tome add` | No confirmation (non-destructive config write). (D-04) |

## CLI Output Styling

All CLI output (non-TUI) uses `console::style()` consistent with the `remove.rs` pattern.

| Element | Style |
|---------|-------|
| Directory/skill names in output | `style(name).cyan()` |
| Counts/numbers | `style(n).bold()` |
| Success prefix | `style("Added"/"Reassigned"/"Forked").green()` |
| Error prefix | `eprintln!("error: {e:#}")` (anyhow debug format) |
| Dry-run prefix | `style("Would").yellow()` |
| Confirmation prompts | `dialoguer::Confirm` with default `false` |

## Theme Struct Contract

The `browse/theme.rs` module exports a single `Theme` struct used by all rendering code. No hardcoded `Color::*` values in `ui.rs` after this phase.

```
Theme {
    accent: Color,           // Cyan family — headers, badges, selection indicators
    alert: Color,            // Yellow family — group headers, search display, match highlights
    muted: Color,            // Gray family — hints, inactive text, metadata labels
    selected_bg: Color,      // Dark gray family — selected row background
    status_bar_bg: Color,    // Dark gray family — status bar background
    status_bar_fg: Color,    // Gray family — status bar text
    destructive: Color,      // Red family — errors
    success: Color,          // Green family — confirmations
    code_fg: Color,          // Magenta family — inline code
    preview_header: Style,   // Bold + accent fg
    preview_bold: Style,     // Bold modifier only
    preview_italic: Style,   // Italic modifier only
    preview_code: Style,     // code_fg color
    match_highlight: Style,  // alert fg + Bold
    group_header: Style,     // alert fg + Bold
}
```

Construction: `Theme::detect()` checks `$COLORFGBG`, returns `Theme::dark()` or `Theme::light()`.
