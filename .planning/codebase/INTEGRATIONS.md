# External Integrations

**Analysis Date:** 2026-04-05

## APIs & External Services

**No remote APIs consumed.** Tome is an offline-first CLI tool with no HTTP dependencies. The codebase contains no `reqwest`, `http`, or similar HTTP client crates.

All integrations are with **local files and directories** on the user's machine.

## Data Storage

**File-Based Persistence:**

All state is stored locally in the user's home directory. No databases or external storage.

**Configuration:**
- `~/.tome/tome.toml` — Main configuration (TOML)
  - Defines skill sources, distribution targets, backup settings
  - Supports tilde expansion (`~`) for path expansion in all fields
  - Loaded via `Config::load_or_default()` in `crates/tome/src/config.rs`

**Library State:**
- `~/.tome/` — Tome home directory (library root and manifest location)
  - Contains consolidated skill library (symlinks + copied files)
  - Backed by atomic temp+rename writes for consistency

**Manifest:**
- `~/.tome/.tome-manifest.json` — Provenance tracker (JSON)
  - Tracks each skill's source path, content hash (SHA-256), source name, sync timestamp
  - Enables idempotent updates: unchanged skills skipped, changed skills re-synced
  - Structure: `{ skills: { "skill-name": { source_path, source_name, content_hash, synced_at, managed } } }`
  - Managed in `crates/tome/src/manifest.rs`

**Lockfile:**
- `~/.tome/tome.lock` — Reproducibility snapshot (JSON)
  - Generated after each `tome sync`
  - Records: schema version, skill entries with source name, content hash, registry ID (if managed plugin), version, git commit SHA
  - Intended for version control
  - Loaded/generated by `crates/tome/src/lockfile.rs`

**Per-Machine Preferences:**
- `~/.config/tome/machine.toml` — Machine-specific disables (TOML)
  - Lists skills disabled on this machine: `disabled = ["skill1", "skill2"]`
  - Lists targets disabled on this machine: `disabled_targets = ["target1"]`
  - Atomic writes via temp+rename pattern in `crates/tome/src/machine.rs`

**Backup:**
- `.tome-backup/` — Git-backed snapshots (optional, controlled by `[backup]` config)
  - Disabled by default in production (`auto_snapshot = false`)
  - When enabled, captures library state snapshots in git history
  - Configured in `crates/tome/src/backup.rs` and `crates/tome/src/config.rs`

## Local Data Sources

**Skill Sources:**

Tome discovers skills from two local source types (configured in `tome.toml`):

1. **ClaudePlugins** — macOS Claude Desktop plugin registry
   - Reads `~/Library/Application Support/Claude/installed_plugins.json`
   - Parses to extract plugin paths and metadata
   - Implementation: `SourceType::ClaudePlugins` in `crates/tome/src/discover.rs`

2. **Directory** — Custom skill directories
   - Flat `walkdir` scan for `*/SKILL.md` files
   - Each SKILL.md becomes a skill (frontmatter + body)
   - Implementation: `SourceType::Directory` in `crates/tome/src/discover.rs`

**Frontmatter Format:**

Each skill is a directory containing `SKILL.md` with YAML frontmatter:

```yaml
---
name: skill-name
description: "What this skill does"
license: MIT
compatibility: "1.0+"
metadata:
  custom-field: value
user-invocable: true
argument-hint: "optional args"
context: "context for skill execution"
agent: "Agent name"
---
# Body content follows...
```

Parsed in `crates/tome/src/skill.rs` via `serde_yaml`.

## Distribution Targets

Targets are data-driven and configurable (no code changes needed to add new tools).

**Configuration Format:**

```toml
[targets.target-name]
enabled = true
method = "symlink"
skills_dir = "/path/to/target/skills"
```

**Current Methods:**

- `symlink` — Symlinks from target's skills directory into `~/.tome/` library

**Known Targets:**

Common targets auto-discovered by wizard (see `KNOWN_TARGETS` in `crates/tome/src/wizard.rs`):
- Claude Desktop (`claude`) → `~/Library/Application Support/Claude/skills`
- Antigravity (`antigravity`) → platform-specific
- Copilot (`copilot`)
- Cursor (`cursor`)
- Windsurf (`windsurf`)
- Others via manual configuration

## File System Conventions

**Skill Directory Structure:**

```
skill-name/
├── SKILL.md           (mandatory: name, description, optional frontmatter)
├── README.md          (optional: docs)
├── other files...     (resources, examples, etc.)
```

**Library Layout:**

```
~/.tome/
├── library/           (consolidated skills, symlinks + copies)
│   ├── skill-1/       (symlink to managed skill OR copy of local skill)
│   ├── skill-2/
│   └── ...
├── .tome-manifest.json (state tracking)
├── tome.lock          (reproducibility)
└── .tome-backup/      (optional git snapshots)
```

## Authentication & Identity

**Not applicable.** Tome requires no authentication, API keys, or identity verification.

All operations are file-based on the local system. No user identity or credentials needed.

## CI/CD & Deployment

**Hosting:**

- GitHub repository: `github.com/martinP7r/tome`
- Binary distribution: Homebrew tap (`martinP7r/homebrew-tap`)
- Release artifacts: GitHub Releases

**CI Pipeline:**

GitHub Actions workflows in `.github/workflows/`:

1. **ci.yml** — Runs on push to main + PRs:
   - Format check (`cargo fmt --check`)
   - Linting (`cargo clippy --all-targets -- -D warnings`)
   - Tests (`cargo test --all`)
   - Release build (`cargo build --release`)
   - Additional checks: `cargo-deny`, `typos`, `cargo-machete`
   - Matrix: `ubuntu-latest`, `macos-latest`

2. **release.yml** — Automated by `cargo-dist` 0.30.3 (auto-generated):
   - Triggers on git tags matching semantic version pattern
   - Builds artifacts for all targets
   - Publishes to GitHub Releases
   - Updates Homebrew formula in tap repository

3. **dependabot.yml** — Dependency updates:
   - Auto-updates cargo dependencies
   - Creates PRs for review

## Backup & Snapshots

**Git-Backed Snapshots (Optional):**

When enabled in config:

```toml
[backup]
enabled = true
auto_snapshot = false  # Manual snapshots only (production default)
```

- `.tome-backup/` directory in library
- Each snapshot is a git commit capturing library state
- Useful for audit trails and recovery
- Implementation: `crates/tome/src/backup.rs`

## Disaster Recovery

**Idempotent Sync:**

The manifest + lockfile design enables safe recovery:

1. Delete library: `rm -rf ~/.tome/library/`
2. Run `tome sync` — rebuilds library from sources using manifest hashes
3. Unchanged skills remain unchanged (compared via SHA-256)

**Source of Truth:**

- For managed skills: original package manager location (`installed_plugins.json`, etc.)
- For local skills: configured directory sources in `tome.toml`
- Library is derivative; always rebuildable from sources
