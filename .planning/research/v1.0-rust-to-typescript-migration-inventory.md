# tome — Feature Inventory for Rust → TypeScript Migration

> Snapshot taken on 2026-04-27 against `main` at commit `83751a1` (v0.8.2). This is the
> *complete* surface area that a TS rewrite would have to reproduce to claim feature
> parity. It is intentionally exhaustive — the goal is to make hidden complexity visible
> so cost estimates are not based on the user-facing command list alone.

| Metric | Value | Notes |
|---|---|---|

## 3. Module-by-module breakdown

LOC counts are source only (tests excluded). "TS notes" call out non-trivial translation
issues — most cost overruns will live in these.

| Module | LOC | Responsibility | TS notes |
|---|---:|---|---|
| `config.rs` | 2,231 | TOML schema, `Config`, `DirectoryConfig`, `DirectoryName`, `DirectoryType`, `DirectoryRole`, deserialize-time validation, tilde expansion. | Replace `serde` derives with `zod` or `valibot` schemas. Newtype semantics (`DirectoryName`) become branded types — runtime checks must be added explicitly. The largest single file by far. |
| `lib.rs` | 1,829 | `run()` dispatcher + full `sync` pipeline orchestration. | Rewrite as a top-level command router; lots of glue, low algorithmic content. |
| `wizard.rs` | 1,820 | Interactive `tome init`: auto-discovers known directory locations, prompts via `dialoguer` (MultiSelect/Input/Confirm/Select), writes config. | Rewrite against `@clack/prompts` or `inquirer`. Snapshot tests rely on deterministic prompt output — needs equivalent harness. |
| `library.rs` | 1,350 | Consolidate step. Two strategies: managed (symlink library→source) and local (copy into library). Manifest-driven idempotency via SHA-256. Repairs stale state. | Hash logic is portable (`node:crypto`). Symlink semantics on macOS/Linux must be tested explicitly. |
| `doctor.rs` | 1,170 | Diagnose orphans, missing manifest entries, broken legacy symlinks; interactive per-item repair. | Lots of branching — straightforward but tedious port. |
| `discover.rs` | 1,028 | Two source scanners: `ClaudePlugins` (`installed_plugins.json` v1 + v2 formats) and `Directory` (flat walkdir). First-source-wins conflict rules + exclusion list. | Replace `walkdir` with `fast-glob`/`globby`. JSON schema variants must be preserved exactly. |
| `relocate.rs` | 993 | Move library to a new path, rewrite all target symlinks atomically. | Atomic move semantics across volumes (`EXDEV`) need explicit handling on Node. |
| `lockfile.rs` | 832 | Generate/load `tome.lock`, atomic temp+rename write. Records name, hash, source, registry id, version, git commit SHA. | TOML serializer ordering matters for stable diffs — pin a deterministic writer. |
| `status.rs` | 673 | Single-pass directory scan; tabled or JSON output. | Replace `tabled` with `cli-table3` or hand-rolled formatter. |
| `remove.rs` | 623 | Plan / render / execute removal of a directory entry. | Pure logic. |
| `distribute.rs` | 622 | Symlink library → target; circular-symlink detection (`shares_tool_root`). | Symlink edge cases (target dir is itself inside library) need careful tests. |
| `backup.rs` | 577 | Five subcommands shelling out to `git`. | Use `simple-git` or just `execa`. Identical output formatting required. |
| `lint.rs` | 516 | Validate SKILL.md frontmatter (YAML), name conventions, description length, Unicode tag codepoints. CI-friendly exit 1. | Replace `serde_yaml` with `js-yaml` or `yaml`. Unicode rules must be ported verbatim. |
| `machine.rs` | 475 | `~/.config/tome/machine.toml`: `disabled`, `disabled_directories`, per-directory `enabled`/`disabled` skill lists. Atomic writes. | Two filesystems (`~/.tome/` portable + `~/.config/tome/` machine-local) must remain split. |
| `cleanup.rs` | 468 | Remove stale library entries, broken target symlinks, disabled-skill symlinks. Verifies symlinks point into the library before removing. | Verification is the safety property — port carefully. |
| `manifest.rs` | 407 | `.tome-manifest.json`: per-skill provenance, content hashes, sync timestamps. `hash_directory()` deterministic SHA-256. | Hash determinism (sort entries, byte order) is critical for idempotency. |
| `reassign.rs` | 346 | `reassign` and `fork` (plan/render/execute). | Pure logic. |
| `update.rs` | 340 | Lockfile diff + interactive triage of new/changed/removed skills. | Diff algorithm is small; UX flow is the work. |
| `paths.rs` | 322 | `TomePaths` bundle (`tome_home`, `library_dir`, `config_dir`); symlink resolution helpers; tilde expansion; absolute path validation. | Replicates a Rust newtype pattern — branded types in TS, plus runtime guards. |
| `install.rs` | 311 | Shell completion generation/installation for bash/zsh/fish/powershell. | Replace `clap_complete` output with hand-written completion templates per shell — this is *not* a free generator like clap. |
| `git.rs` | 290 | Shallow clone to `~/.tome/repos/<sha256>/`, ref pinning (branch/tag/rev), HEAD SHA reading. | Use `simple-git` or shell out to `git`. |
| `add.rs` | 294 | `tome add <url>`: parse URL, generate name, write directory entry. | Pure logic. |
| `cli.rs` | 260 | `clap` CLI definition. | Rewrite against `commander` or `yargs`. Loss of `clap`'s automatic help/error UX — non-trivial to match. |
| `eject.rs` | 239 | Remove tome's symlinks from all targets. | Pure logic. |
| `validation.rs` | 167 | `validate_identifier()` (rejects empty, `.`, `..`, whitespace, path separators); SkillName/DirectoryName invariants. | Centralised — port once, reuse. |
| `skill.rs` | 124 | SKILL.md parsing helpers (frontmatter + body split). | Small. |
| `main.rs` | 27 | Entry-point shim. | Trivial. |

### 3.1 The browse TUI in detail

| File | LOC | Notes |
|---|---:|---|
| `browse/app.rs` | 957 | App state machine (`Mode::{Normal,Search,Detail,Help}`), `SortMode`, `DetailAction`, `StatusMessage` toasts, all key bindings. |
| `browse/ui.rs` | 514 | `ratatui` rendering: list, preview pane, scrollbar, status bar, help overlay, fuzzy match highlighting. |
| `browse/markdown.rs` | 243 | In-pane markdown preview rendering. |
| `browse/fuzzy.rs` | 199 | `nucleo-matcher` integration + result ranking. |
| `browse/theme.rs` | 167 | Adaptive dark/light theme detection. |
| `browse/mod.rs` | 60 | Crossterm event loop + bootstrapping. |

The TS equivalent will be `ink` (React-based) or `blessed`/`neo-blessed`. Neither is a
drop-in for `ratatui`; the rendering model is different (ink uses Yoga/flexbox,
ratatui uses constraint solving). Expect ~50% more code than the Rust version, and
plan to redesign rather than transliterate.

## 4. Cross-cutting concerns

These appear in many modules and shape the overall shape of the TS port.

| Concern | Rust mechanism | TS equivalent |
|---|---|---|
| Type-safe identifiers | Newtypes (`SkillName`, `DirectoryName`, `ContentHash`) with custom `Deserialize` validating at parse time. | Branded types + zod refinements. Runtime cost added; compile-time guarantees weaker. |
| Exhaustive dispatch | `match` on enums (compiler-checked). | Discriminated unions + `assertNever`. Works well, but discipline-dependent. |
| Atomic writes | `tempfile` + rename. | Hand-roll `fs.writeFile(tmp)` + `fs.rename`. Watch `EXDEV`. |
| Idempotency | SHA-256 of directory contents (`hash_directory`) → manifest. | `node:crypto` `createHash('sha256')`. Determinism requires explicit sort + byte order. |
| `dry_run` threading | Boolean flag passed through every mutating function; counts the same either way. | Same pattern; lots of mechanical plumbing. |
| Tilde expansion | `~` resolved to `$HOME` everywhere a path is read. | Manual; `os.homedir()` + replace. |
| Symlink utilities | `paths::shares_tool_root`, circular-symlink detection. | Port verbatim. |
| Errors | `anyhow::Result` + `.context()` strings shown on stderr as `error: <chain>`. | Custom error wrapper that carries a context chain; or `verror`/`ts-results` style. |
| TTY detection | `IsTerminal` + interactive prompts only when TTY. | `process.stdout.isTTY`. |
| Logging / verbose | `--verbose` gates extra writes; spinners via `indicatif`. | `ora` or `cli-progress`. |

## 5. File formats (must be byte-stable across versions)

| File | Format | Stability requirement |
|---|---|---|
| `~/.tome/tome.toml` | TOML, schema in `config.rs`. | Round-trip through TS must produce identical bytes for unchanged input (or migrations break). |
| `~/.config/tome/machine.toml` | TOML. | Same. |
| `~/.tome/.tome-manifest.json` | JSON, sorted keys, atomic writes. | Hash determinism depends on this. |
| `~/.tome/tome.lock` | TOML, atomic writes. | Diffability across syncs depends on stable ordering. |
| `<library>/.gitignore` | Auto-generated. | Must match Rust output exactly for clean diffs. |

## 8. Distribution & release

The Rust project ships single static binaries via `cargo-dist` with Homebrew + GitHub
Releases. A TS port has to choose between three distribution paths, each with
implications:

| Path | Pros | Cons |
|---|---|---|
| `npm` package + `npx tome` | Simplest; works everywhere Node runs. | Requires Node on user machines; cold-start latency. |
| `pkg` / `nexe` / `bun build --compile` | Single binary like today. | Bigger binaries (~50–80 MB); platform builds still required; bun excludes some Node APIs. |
| Deno compile | Modern runtime, single binary, secure-by-default. | Smaller ecosystem; some npm packages don't work. |

Bun `--compile` is currently the closest match to today's UX, but adds a runtime
dependency for the project's own development.

## 9. Constraints inherited from current architecture

The TS port has to preserve these, or break user expectations:

- **Unix-only** (POSIX symlinks). Windows is out of scope.
- **Two-tier model**: discovery dirs → library → distribution dirs. Library is the
  source of truth.
- **Managed vs local strategy split** during consolidation.
- **First-source-wins** name-conflict rule.
- **Atomic temp+rename** for every config/lockfile/manifest write.
- **`dry_run`** threading through every mutating function.
- **No nested git**: git source clones live in `~/.tome/repos/`, not inside the library.
- **No backward compat with mid-version configs** — the project explicitly opts out of
  migration tooling because there is a single user.

## 10. Out-of-scope for this inventory

These are real costs but not feature parity:
- Performance regressions (TS will be 5–50× slower on hashing/scan hot paths; needs
  benchmarking on realistic library sizes).
- Editor tooling (`rust-analyzer` workflow → `tsc`/`eslint`/`vitest` workflow).
- Long-tail maintenance differential (npm dependency churn vs Cargo).
- Loss of compile-time guarantees (memory safety, exhaustive matching).

- **27 source files**, **~19.2k LOC** to translate.
- **~7k LOC of tests, 608 tests** to re-implement.
- **14 commands + 5 subcommands**, all with stable output contracts.
- **5 file formats** that need byte-stable round-trip.
- **2 high-risk modules** (`browse/` TUI, `wizard.rs`) that should be redesigned, not
